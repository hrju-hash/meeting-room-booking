# 🔥 Firebase 설정 - 완전 자동화 가이드

## 🚀 가장 쉬운 방법 (3분 완료)

### 1. 자동 설정 페이지 열기
```
auto-firebase-setup.html 파일을 브라우저에서 열기
```

### 2. 단계별로 버튼만 클릭
- 각 단계의 버튼을 클릭하면 자동으로 필요한 페이지가 열립니다
- 화면의 안내를 따라 진행하세요

### 3. 구성 정보 저장 및 배포
- 구성 정보를 붙여넣고 "저장 및 자동 배포" 클릭
- 다운로드된 `firebase-config.js` 파일을 프로젝트 폴더에 넣기
- `setup-and-deploy.bat` 파일 실행

---

## 📋 상세 단계

### 1단계: Firebase 프로젝트 생성
1. "Firebase 콘솔 열기" 버튼 클릭
2. "프로젝트 추가" 클릭
3. 프로젝트 이름: **malgn-booking** 입력
4. Google Analytics: **비활성화**
5. "프로젝트 만들기" 클릭

### 2단계: Realtime Database 생성
1. "데이터베이스 설정 가이드 열기" 버튼 클릭
2. "데이터베이스 만들기" 클릭
3. 위치: **asia-northeast3 (Seoul)** 선택
4. 보안 규칙: **테스트 모드로 시작** 선택
5. "규칙" 탭에서 아래 코드로 교체 후 "게시":
```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

### 3단계: 웹 앱 등록
1. "웹 앱 설정 가이드 열기" 버튼 클릭
2. 웹 아이콘 (`</>`) 클릭
3. 앱 닉네임: **malgn-booking-web** 입력
4. "Firebase Hosting도 설정하시겠습니까?" → **체크 해제**
5. "앱 등록" 클릭
6. 표시되는 `firebaseConfig` 객체 **전체 복사**

### 4단계: 구성 정보 저장
1. 복사한 구성 정보를 입력란에 붙여넣기
2. "저장 및 자동 배포" 버튼 클릭
3. 다운로드된 `firebase-config.js` 파일을 프로젝트 폴더에 넣기
4. `setup-and-deploy.bat` 파일 실행

## ✅ 완료 확인

1. 브라우저에서 `index.html` 열기
2. F12 > Console 탭
3. "Firebase가 성공적으로 초기화되었습니다." 메시지 확인 ✅
4. 회의실 예약하기
5. 새로고침 (F5)
6. 예약이 그대로 남아있는지 확인 ✅

## 🆘 문제 해결

### "Firebase가 초기화되지 않았습니다"
- `firebase-config.js` 파일이 프로젝트 폴더에 있는지 확인
- 구성 정보가 올바른지 확인
- 브라우저 콘솔에서 오류 메시지 확인

### 데이터가 저장되지 않아요
- Firebase 콘솔 > Realtime Database > 규칙 탭 확인
- 읽기/쓰기가 모두 `true`인지 확인

### 새로고침 후 데이터가 사라져요
- Firebase가 제대로 초기화되었는지 확인
- Firebase 콘솔에서 데이터가 저장되어 있는지 확인
